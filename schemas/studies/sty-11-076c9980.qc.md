## ENVO labels missing in NMDC ingest (terms themselves are fine)

All three environmental context CURIEs resolve to semantically appropriate ENVO terms, but NMDC's `*_term_name` columns are NULL — the labels were never populated at ingest:

| Field | Term ID | Actual ENVO label | NMDC `*_term_name` | NMDC `*_has_raw_value` |
|---|---|---|---|---|
| `env_medium_term_id` | `ENVO:00005778` | tropical soil | NULL | `ENVO:00005778` |
| `env_broad_scale_term_id` | `ENVO:01000174` | forest biome | NULL | `ENVO:01000174` |
| `env_local_scale_term_id` | `ENVO:01001803` | tropical forest | NULL | `ENVO:01001803` |

All three are good ENVO choices for a Luquillo Experimental Forest tropical-soil enrichment study. The issue is purely that NMDC stored the bare CURIE in `*_has_raw_value` (where you'd normally expect e.g., `"tropical soil [ENVO:00005778]"`) and left `*_term_name` empty. **Fix: backfill the labels from ENVO**, no re-curation needed.

This is a different failure mode from `sty-11-vh2hty57` (where the env_medium term resolves but is semantically wrong — claiming bulk soil for rhizosphere samples). Here the terms are correct; only the label-population step in the ingest pipeline failed.

> Audit-process note: my first pass mistakenly concluded these IDs were obsolete because the MCP OLS search/fetch tools returned empty results for `ENVO:00005778`. The terms exist; the MCP tool's CURIE-based lookup is unreliable. Direct OLS v2 API by IRI confirms all three are current.

## Experimental design lives in free text

The `description` column carries the experimental design as unstructured text. Three examples from the data:

```
"Lab enrichment of tropical soil microbial communities from Luquillo Experimental Forest,
 control, no incubation, no C amendment, aerobic conditions"

"Lab enrichment of tropical soil microbial communities from Luquillo Experimental Forest,
 44 days of incubation with ryegrass amendment (10% of native soil C), fluctuating
 4 days anoxic (N2 gas) and 4 days oxic (21% O2) regime"

"Lab enrichment of tropical soil microbial communities from Luquillo Experimental Forest, ..."
```

Recoverable factors (would need parsing):
- treatment type: control vs amended
- incubation duration (days, e.g. `44`)
- C amendment identity (e.g. `ryegrass`)
- C amendment dose (e.g. `10% of native soil C`)
- redox regime (e.g. `aerobic`, `anoxic`, `fluctuating 4d/4d`)
- O2 / N2 atmosphere composition

None are surfaced as structured fields. The bermap profile's `redox_fluctuation_treatment` slot is the only structured stand-in; it has no source column.

## Lab microcosm vs source ecosystem

`sample_collection_site = "Lab microcosm"` is populated for all 105 samples. These are lab enrichments derived from Luquillo tropical-forest soils.

`env_local_scale = ENVO:01001803` (tropical forest) describes the **source ecosystem**, not the lab microcosm. That matches the MIxS convention for derived/enrichment samples (env_* describes the field origin), and the lab-vs-field distinction is carried separately by `sample_collection_site`. So this is *not* a semantic error — but it does mean any analysis filtering on env_local_scale alone will treat these as field samples, which they aren't. Filter on `sample_collection_site` to recover the distinction.
