But the choice is internally consistent and arguably defensible for a lab study with no field component.

## EcoFAB multi-lab design

Sample names encode lab + replicate:

```
LabA_01_Axe_1
LabA_02_Axe_2
LabA_03_Axe_3
```

- `LabA` = participating lab identifier
- `01` / `02` / `03` = sample within lab
- `Axe` = (likely a Brachypodium or other host genotype identifier or condition code)
- `_N` = replicate

This is a **multi-lab reproducibility study** — the design factor "participating lab" is critical to interpretation. None of `samp_name`-parsable factors (lab, replicate, host condition) are surfaced as structured NMDC fields.

Per the description, this is a **global collaborative effort** using EcoFAB 2.0 plant growth devices to study root microbiome assembly in a standardized way across multiple laboratories. Reproducibility comparisons fundamentally require the lab identifier to be a first-class structured variable.

## Agent curation pass (2026-06-19)

Full `/ncbi-to-nmdc` workflow run against PRJNA1151037 (175 biosamples; 163 DataGenerations — 12 mock samples have no seq data). This is the only study in the cohort with **`validator_rejected` outcomes** — and they're principled, not failures.

| Slot | predicted | resolved_from_raw | resolved_at_pipeline | left_sentinel | validator_rejected |
|---|---:|---:|---:|---:|---:|
| `env_broad_scale` | 0 | 0 | 0 | 0 | **175** |
| `env_local_scale` | 0 | 0 | **105** | 0 | **70** |
| `env_medium` | 0 | 0 | **175** | 0 | 0 |

- `env_broad_scale` rejected for all 175 — pipeline's `ENVO:01001001 plant-associated environment` isn't a biome. **There is no good ENVO biome term for an indoor EcoFAB microcosm setup.** Curator-follow-up backlog.
- `env_local_scale`:
  - 105 root samples → kept rhizosphere `ENVO:00005801`
  - 70 sterile-water mocks → rejected `ENVO:01000621 microcosm` (under manufactured product, not astronomical body part)
- `env_medium` accepted for all 175:
  - 105 root → `ENVO:01000349 root matter`
  - 70 mocks → `ENVO:00005791 sterile water`

Caveat: 70 sterile-water mocks carry `samp_taxon_id = NCBITaxon:1118232 root metagenome` from the pipeline — semantically wrong (sterile water isn't a root metagenome) but outside the curation report scope.

## Open items

- **Participating lab is the primary independent variable** in a multi-lab reproducibility study but lives only in `samp_name`. Needs a structured slot in the bermap profile and population from `samp_name`.
- Bermap profile's `LaboratorySiteEnum` ("four participating laboratories") and `EcofabGrowthSystemEnum` ("EcoFAB") are placeholders — should be reified with actual enum values once lab identifiers are confirmed
- The 23-sample gap between NMDC (140) and NCBI (163) likely accounts for mock-community standards / blank controls (same pattern as `sty-11-wbc14h22`); not verified per-sample
